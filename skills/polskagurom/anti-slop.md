# Polski anti-slop — typowe AI-błędy

Materiał operacyjny do `/polskagurom`. Format: PRZED → PO. Bez wyjaśnień, tylko konkrety.

---

## 1. Kalki z angielskiego

### Zwroty biznesowo-techniczne
- "w oparciu o dane" → **na podstawie danych** / **z danych**
- "dedykowany zespół" → **osobny zespół** / **wyznaczony zespół**
- "dedykowane rozwiązanie" → **rozwiązanie szyte na miarę** / **specjalnie dla X**
- "adresować problem" → **zająć się problemem** / **odpowiedzieć na problem** / **rozwiązać**
- "adresować potrzeby" → **odpowiadać na potrzeby**
- "dostarczać wartość" → **dawać coś od siebie** / **być przydatnym** / **robić robotę**
- "dostarczać rozwiązań" → **rozwiązywać** / **dawać rozwiązania**
- "user journey" → **ścieżka użytkownika**
- "deep dive w temat" → **wgryźć się w temat** / **rozkminić**
- "insights" → **wnioski** / **obserwacje** / **co z tego wynika**
- "case study" → **przykład** / **historia**
- "leverage'ować coś" / "lewarować" → **wykorzystać**

### Czasowniki-kalki
- "kontaktować kogoś" → **skontaktować się z kimś**
- "aplikować zmianę" → **wprowadzić zmianę** / **zastosować**
- "implementować" (poza kodem) → **wdrożyć** / **wprowadzić**
- "wspierać format" → **obsługiwać format**
- "evaluować opcje" → **ocenić opcje** / **przeanalizować**
- "performować dobrze" → **działać dobrze** / **radzić sobie**
- "ekspandować na rynek" → **wejść na rynek**
- "targetować grupę" → **kierować do grupy** / **trafiać w grupę**
- "fokusować się na" → **skupiać się na**

### Rzeczowniki-kalki
- "challenge" → **wyzwanie** (a lepiej: **problem** / **trudność**)
- "feedback" → **opinia** / **uwagi** / **co o tym myślisz**
- "framework do myślenia" → **schemat myślenia** / **sposób patrzenia**
- "mindset" → **nastawienie** / **podejście**

### Predykatywy
- "to jest game changer" → **to zmienia wszystko** / **to przełom**
- "low-hanging fruit" → **najłatwiejsze do zgarnięcia** / **co leży na wierzchu**
- "no-brainer" → **oczywista sprawa** / **bez gadania**
- "ma sens robić X" → **warto zrobić X** / **dobrze zrobić X**

## 2. Nadużywane słowa AI (charakterystyczne "tells")

### Konektory i wypełniacze
- "Ponadto, ..." → **Do tego** / **Poza tym** / (wytnij)
- "Warto zauważyć, że ..." → (wytnij, zostaw fakt) / **Ważne:**
- "Należy podkreślić, że ..." → **Ważne:** / (wytnij)
- "Niemniej jednak" → **Mimo to** / **Ale**
- "W kontekście X" → **Jeśli chodzi o X** / **Przy X** / **Co do X**
- "W szczególności" → **Zwłaszcza** / **Głównie**
- "Z drugiej strony" (gdy nie było pierwszej) → **Ale** / (wytnij)
- "Co więcej" → **Do tego** / (wytnij)
- "W rezultacie" → **Dlatego** / **Stąd**

### "Wytrychy" oceniające
- "kluczowy aspekt" → **ważne** / (konkretnie co)
- "istotny element" → **ważne** / (konkretnie co)
- "znaczący wpływ" → **mocno wpływa** / **dużo zmienia**
- "fundamentalna rola" → **podstawa**
- "holistyczne podejście" → **całościowe** / **na wszystko naraz**
- "kompleksowe rozwiązanie" → **wszystko w jednym** / (konkretnie co obejmuje)
- "synergiczne efekty" → (wytnij — to nic nie znaczy)
- "transformacyjna zmiana" → **duża zmiana** / **przełom**

### Czasowniki-puste
- "stanowi" → **jest** ("X stanowi przykład" → "X to przykład")
- "posiadać" → **mieć** ("posiadać doświadczenie" → "mieć doświadczenie")
- "dokonać X" → czasownik prosty:
  - "dokonać analizy" → **przeanalizować**
  - "dokonać wyboru" → **wybrać**
- "ulec zmianie" → **zmienić się**
- "podlegać ocenie" → **być oceniane** / **oceniamy**
- "znajduje zastosowanie" → **stosuje się** / **używa się**

### Zakończenia akapitów (typowe AI-outro do wycięcia)
- "Podsumowując, ..." → (wytnij cały akapit)
- "Reasumując, ..." → (wytnij)
- "W efekcie końcowym ..." → **W efekcie** / **Stąd**
- "Mam nadzieję, że ten przewodnik okaże się pomocny" → (wytnij)
- "Pamiętaj, że ..." (moralizujące zakończenie) → (wytnij)

## 3. Złe konstrukcje gramatyczne

### Strona bierna (AI ją uwielbia, polski nie)
- "Decyzja została podjęta przez zespół" → **Zespół zdecydował**
- "Plik został zapisany" → **Zapisałem plik** / **Plik się zapisał**
- "Zostały przeprowadzone testy" → **Przetestowaliśmy** / **Zrobiliśmy testy**
- "Email został wysłany" → **Wysłałem email** / **Email poszedł**

### Łańcuchy dopełniaczy (więcej niż 2 = popraw)
- "proces analizy danych użytkowników aplikacji" → **analizujemy, jak użytkownicy korzystają z aplikacji**
- "system zarządzania treścią serwisu klienta" → **CMS dla serwisu klienta**
- "raport wyników badania satysfakcji uczestników szkolenia" → **raport: jak uczestnicy ocenili szkolenie**

### Nominalizacje (rzeczownik zamiast czasownika)
- "dokonanie analizy" → **przeanalizować**
- "podjęcie decyzji" → **zdecydować**
- "realizacja projektu" → **zrobić projekt** / **dowieźć projekt**
- "wdrożenie zmian" → **wdrożyć zmiany**
- "przeprowadzenie spotkania" → **spotkać się**
- "w celu zwiększenia sprzedaży" → **żeby sprzedać więcej**

### Błędy w przypadkach po przyimkach
- "dyskutować temat" → **dyskutować o temacie** / **rozmawiać o**
- "bazować o coś" → **bazować na czymś** / **opierać się na**
- "udać się celem zrobienia" → **żeby zrobić**

## 4. Szyk zdania

- "Ja uważam, że to działa" → **Uważam, że to działa** (pomiń zbędne "ja")
- "My zrobiliśmy to wczoraj" → **Zrobiliśmy to wczoraj**
- "Nigdy nie powinieneś nie sprawdzać" (podwójna negacja ang.) → **Zawsze sprawdzaj**
- Reguła "się": trzyma się czasownika, nie wyrzucaj go na koniec.
- Orzeczenie na końcu (kalka z DE/EN złożeń) → przepisz prościej.

---

## 5. Interpunkcja

### Przecinki przed "że/który/jeśli/gdy/bo/aby" — AI często gubi
- "Wiem że to działa" → **Wiem, że to działa**
- "Plik który zapisałem" → **Plik, który zapisałem**
- "Zrób to jeśli zdążysz" → **Zrób to, jeśli zdążysz**

### Brak przecinka po imiesłowowym równoważniku
- "Robiąc to musisz uważać" → **Robiąc to, musisz uważać**

### Myślnik vs półpauza vs łącznik
- Łącznik `-` tylko w wyrazach złożonych: **biało-czerwony**
- Półpauza `–` w zakresach: **2024–2026**
- Myślnik `—` (em dash, ze spacjami) jako wtrącenie: **To jest — moim zdaniem — błąd**
- AI często wstawia `-` zamiast `—`. Popraw.

### Cudzysłów polski
- AI używa `"..."` (proste) → polski standard to **„..."**

### Wykrzykniki i wielokropki
- AI nadużywa "!" — jeden wystarczy, najczęściej żaden.

## 6. Pompatyczność i redundancje

### Pleonazmy
- "w pełni kompletny" → **kompletny**
- "okres czasu" → **okres** / **czas**
- "cofnąć się do tyłu" → **cofnąć się**
- "spadać w dół" → **spadać**
- "kontynuować dalej" → **kontynuować**
- "potencjalna możliwość" → **możliwość**
- "dzień dzisiejszy" → **dziś, dzisiaj**
- "w miesiącu maju" → **w maju**

### Hedging i wodolejstwo
- "Można powiedzieć, że to działa" → **To działa**
- "Wydaje się, że jest to dobre rozwiązanie" → **To dobre rozwiązanie**
- "W mojej skromnej opinii" → **Moim zdaniem** / (wytnij)
- "Generalnie rzecz biorąc" → **Ogólnie** / (wytnij)
- "Jak już wcześniej wspomniałem" → (wytnij)

### Korpomowa
- "w zakresie X" → **przy X** / **dla X**
- "celem osiągnięcia" → **żeby osiągnąć**
- "z uwagi na" → **bo** / **przez**
- "w związku z powyższym" → **dlatego** / **stąd**
- "uprzejmie informuję, że" → **piszę, że**

### Intensyfikatory-watę
- "absolutnie krytyczne" → **krytyczne**
- "naprawdę bardzo ważne" → **ważne**
- "ekstremalnie istotny" → **ważny**

## 7. Ortografia, typografia, anglicyzmy

### Wielkie litery
- AI często pisze rzeczowniki pospolite z dużej (z ang.): "Sztuczna Inteligencja", "Marketing Cyfrowy" → **wszystko z małej**
- Dni/miesiące z małej: "Poniedziałek" → **poniedziałek**, "Maj" → **maj**
- Tytuły: tylko pierwsza litera duża: "Jak Skutecznie Zarządzać Czasem" → **Jak skutecznie zarządzać czasem**

### Polskie znaki — gdzie AI gubi
- "moge", "bede", "ktory" → **mogę, będę, który**
- Sprawdź: ą, ę, ć, ł, ń, ó, ś, ź, ż

### Liczby i jednostki
- "5km" → **5 km** (spacja)
- Daty w tekście: **21 maja 2026** lub **21.05.2026**
- Tysiące: **10 000**, nie **10,000**
- Dziesiętne: **3,14** (przecinek)

### Skróty
- "np" → **np.** | "tzn" → **tzn.** | "itd" → **itd.**
